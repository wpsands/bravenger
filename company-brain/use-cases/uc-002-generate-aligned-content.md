# UC-002: Generate Aligned Content from the Brain

## Trigger

A team member needs to produce content. This happens daily, across every discipline:

- A marketer needs a blog post about a specific use case.
- A sales rep needs a discovery prep doc before a call with a technical founder.
- A PMM needs a competitive positioning brief for a new competitor.
- A product manager needs a feature brief that references the right personas and use cases.
- A CS lead needs an onboarding walkthrough for a specific use case.

The trigger is simple: someone needs to say something about the product, the market, or the customer. Before, they guessed or asked the founder. Now, they point an agent at the Brain.

## Current State

Before the Brain, content generation looks like this:

- **AI with no context.** The team member opens Claude or ChatGPT and writes a prompt from scratch. The AI produces something that sounds good but doesn't match the company's positioning, uses the wrong value props, or describes features inaccurately.
- **Copy-paste archaeology.** The team member digs through old blog posts, pitch decks, and Slack messages trying to find the "right" way to describe something. They find three versions. None are current.
- **Founder-as-bottleneck.** The team member writes a draft, sends it to the founder, waits 3 days for feedback, gets told "that's not quite right," rewrites it, and repeats.
- **Inconsistent output.** Five people produce five versions of the same message. The website says one thing. The sales deck says another. The support docs say a third. Nobody notices until a customer does.

The result is slow, inconsistent, and unreliable content production. The team spends more time figuring out what to say than actually saying it.

## Outcome

With the Brain, content generation becomes:

- **Agent reads the Brain first.** Before producing any output, the agent reads the mandatory files: brand-vision.md, icp.md, relevant personas, relevant use cases, AGENTS.md, and the discipline-specific guideline.
- **Output is cited.** Every claim in the output includes a `[Source: filename#section]` citation. The human reviewer can verify any statement by checking the source file.
- **Consistent across teams.** A blog post, a sales email, and a product brief about the same use case all say the same thing — because they all read the same source files.
- **Fast.** Content that took hours of research and founder review now takes minutes. The agent has all the context. The human reviews for accuracy, not for alignment.
- **Maintainable.** When positioning changes, you update the Brain. Every future piece of content reflects the update. No need to retrain people or rewrite templates.

The specific output depends on the discipline:
- **Marketing:** Blog posts, landing pages, ad copy, email campaigns, social posts.
- **Sales:** Discovery prep docs, outbound sequences, proposal language, competitive responses.
- **PMM:** Launch briefs, competitive intel, messaging frameworks, enablement decks.
- **Product:** Feature briefs, persona reaction assessments, adoption scoring models.
- **CS:** Onboarding guides, use case walkthroughs, migration playbooks.

## Gotchas

- **Agent quality depends on governance.** If the agent doesn't follow the read order in AGENTS.md, the output will still drift. Governance is not optional — it's the mechanism that makes this work.
- **Not all content is agent-generated.** Some content requires human judgment, creative writing, or sensitive handling. The Brain informs the human; it doesn't replace them.
- **Outdated Brain = outdated content.** If the Brain isn't maintained, agents produce content based on stale information. The system is only as current as its inputs.
- **Citation verification is essential.** Just because an agent cites a source doesn't mean the citation is correct. Human reviewers should spot-check citations, especially for high-stakes content (website, sales proposals).
- **Persona selection matters.** An agent producing content for the Technical Founder persona will use different language than one targeting the Head of Marketing. Choosing the wrong persona produces technically correct but tonally wrong content.

## Gaps

- **No automated citation validation.** Agents cite sources, but there's no automated check that the citation is accurate. A human must verify.
- **No content approval workflow.** The Brain doesn't include a review/approve/publish pipeline. That lives in the team's existing tools (Notion, Linear, Google Docs).
- **No output quality scoring.** There's no automated way to measure how well an agent's output aligns with the Brain. This is where UC-003 (drift detection) comes in, but it's partially supported today.
- **No multi-format output.** The Brain produces Markdown. Converting to slides, PDFs, or design-ready formats requires external tools.

## Connected Use Cases

- **UC-001: Build a Company Brain.** The Brain must exist before content can be generated from it. UC-001 creates the foundation that UC-002 consumes.
